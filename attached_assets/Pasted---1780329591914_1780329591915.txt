Нужно изменить логику направлений эксперта.

ВАЖНО:
Направление, которое заказчик выбирает при создании заказа, НЕ трогать.
Создание заказа и palata_requests.expertise_direction_id не менять.

Меняем только:
1. регистрацию эксперта;
2. редактирование профиля эксперта.

# Цель

Эксперт НЕ должен выбирать направления экспертизы вручную.

Направления эксперта должны назначаться автоматически на основании введённых номеров сертификатов.

# Что есть в БД

Есть таблица сертификатов:

palata_certificates

Основные поля:
- certificate_number
- expert_full_name
- specialty_code
- valid_from
- valid_to
- is_active

Есть таблица кодов:

palata_specialty_codes

Основные поля:
- code
- expertise_direction_id

Связь:
palata_specialty_codes.expertise_direction_id
→ palata_expertise_directions.id

Есть таблица направлений эксперта:

palata_expert_directions

В неё нужно записывать направления эксперта автоматически.

# Логика регистрации эксперта

В форме регистрации эксперта заменить ручной выбор направлений на блок:

"Сертификаты эксперта"

Эксперт может ввести несколько номеров сертификатов.

Формат:
- отдельные input-поля;
- кнопка "+ Добавить сертификат";
- можно удалить лишний номер.

Примеры номеров:
- № PS 003231
- PS 003231
- 003231

Нужно нормализовать номер перед поиском:
- убрать лишние пробелы;
- привести к единому формату;
- поиск должен находить сертификат независимо от того, ввёл эксперт "№ PS 003231" или "PS 003231".

# Проверка сертификатов

Для каждого введённого номера:

1. Найти запись в palata_certificates по certificate_number.
2. Проверить is_active = true.
3. Проверить valid_to >= current_date.
4. Взять specialty_code.
5. Если specialty_code содержит несколько кодов через запятую — разобрать все.
6. Для каждого кода найти запись в palata_specialty_codes.
7. Получить expertise_direction_id.
8. Записать уникальные expertise_direction_id в palata_expert_directions.

# Если сертификат найден

Показать эксперту:
- сертификат найден;
- срок действия;
- найденные направления.

Например:
"Сертификат подтверждён. Направления: Строительно-техническая экспертиза, Землеустроительная экспертиза."

# Если сертификат не найден

Не давать автоматически назначать направление.

Показать:
"Сертификат не найден. Заявка будет отправлена на ручную проверку."

Профиль эксперта должен получить статус:
requires_review / draft / pending_review — использовать текущую существующую модель статусов, если она есть.

# Если сертификат найден, но код не распознан

Назначить направление:
"Другие экспертизы"

если оно есть в palata_expertise_directions.

# Редактирование профиля эксперта

В профиле эксперта добавить возможность редактировать список сертификатов:

- добавить сертификат;
- удалить сертификат;
- перепроверить сертификаты.

После сохранения:
1. удалить старые записи эксперта в palata_expert_directions;
2. заново рассчитать направления по актуальному списку сертификатов;
3. записать новые направления в palata_expert_directions;
4. убрать дубли направлений.

# Важно

Эксперт НЕ редактирует направления руками.

Направления только отображаются как результат проверки сертификатов.

Источник истины:
- palata_certificates
- palata_specialty_codes
- palata_expert_directions

Не использовать:
- palata_expert_profiles.specializations
- ручной multi-select направлений эксперта.

# Карточка эксперта

В карточке эксперта показывать:

- номера сертификатов;
- сроки действия;
- подтверждён / требует проверки;
- направления, рассчитанные по сертификатам.

# Matching

Matching не менять по смыслу.

Он должен продолжать использовать:

palata_requests.expertise_direction_id
=
palata_expert_directions.expertise_direction_id

То есть после автоматического назначения направлений эксперт должен попадать в подбор по этим направлениям.
