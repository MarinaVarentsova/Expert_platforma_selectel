ШАГ 3. SERVER API ДЛЯ ЗАГРУЗКИ

В artifacts/palata/server.js создать или окончательно доработать endpoint:

POST /api/palata/cert-import

Request:

Authorization: Bearer <access_token>
Content-Type: application/json

{
  "file_name": "registry.xlsx",
  "rows": [
    {
      "certificate_number": "...",
      "expert_full_name": "...",
      "specialty_text": "...",
      "certificate_period": "...",
      "codes": "...",
      "directions": "...",
      "valid_from": "YYYY-MM-DD или null",
      "valid_to": "YYYY-MM-DD или null",
      "certificate_status": "...",
      "load_status": "..."
    }
  ]
}

Endpoint должен:

1. Проверить Bearer token через текущий auth-service.
2. Получить пользователя из public.palata_users.
3. Разрешить импорт только role = 'admin' и is_active = true.
4. Проверить, что file_name передан.
5. Проверить, что rows является массивом.
6. Получить client из существующего pg.Pool.
7. Выполнить BEGIN.
8. Выполнить:

TRUNCATE TABLE public.palata_certificates_import;

9. Вставить все строки нового файла в public.palata_certificates_import.
10. Пустые строки преобразовывать в NULL там, где это допустимо.
11. valid_from и valid_to передавать как date или NULL.
12. Не передавать текст "NULL".
13. Посчитать фактическую статистику принятого файла:
    - total_rows = количество rows;
    - active_count = количество активных сертификатов;
    - expired_count = количество неактивных/просроченных сертификатов;
    - parse_error_count = количество строк с ошибкой даты или обязательных полей;
    - linked_experts_count = 0;
    - unlinked_experts_count = total_rows;
14. Вставить одну запись в public.palata_certificate_import_logs:
    - created_by = id текущего администратора;
    - file_name;
    - total_rows;
    - active_count;
    - expired_count;
    - parse_error_count;
    - linked_experts_count = 0;
    - unlinked_experts_count = total_rows;
    - status = 'ok';
    - error_message = NULL.
15. Выполнить COMMIT.
16. При любой ошибке выполнить ROLLBACK.
17. Вернуть JSON:

{
  "success": true,
  "result": {
    "total": 0,
    "active": 0,
    "expired": 0,
    "parse_errors": 0,
    "linked_experts": 0,
    "unlinked_experts": 0,
    "file_name": "...",
    "created_at": "..."
  }
}

При ошибке вернуть структурированный JSON:

{
  "success": false,
  "error": "CERT_IMPORT_FAILED",
  "message": "..."
}

Не возвращать и не логировать пароль, токен или PALATA_DATABASE_URL.
