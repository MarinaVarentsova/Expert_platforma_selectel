Следующий этап: реализовать автоматический подбор экспертов + Action Center «Требуют действия».

Сейчас метрики закрыты, но логики подбора пока нет. Нужно сделать связку:

`создан заказ → система подобрала экспертов → заказчик видит действие “Выбрать эксперта” → эксперт получает действие “Вас выбрали”`

Канбан НЕ удалять. Канбан остаётся как обзор.
Action Center — дополнительная вкладка для задач, требующих решения.

# 1. Автоматический подбор экспертов

После создания заказа или при ручном запуске подбора система должна искать экспертов.

## Фильтры подбора

Брать только экспертов:

* профиль активен;
* принимает заказы;
* направление заказа совпадает со специализациями эксперта;
* эксперт не отказался ранее по этому заказу;
* если регион совпадает — высокий приоритет;
* если регион не совпадает, эксперт может участвовать только если готов к командировкам;
* если заказ дистанционный — регион не должен быть жёстким фильтром.

## Приоритеты сортировки

Сортировать экспертов по score:

1. совпадение направления;
2. совпадение региона;
3. готовность к командировкам;
4. регистрация в Палата судебных экспертов РФ;
5. регистрация в Центр судэксперт;
6. рейтинг эксперта;
7. количество выполненных заказов;
8. низкий процент отказов.

## Результат подбора

* выбрать до 5 экспертов;
* записать их в `palata_request_matches`;
* статус заказа перевести в `expert_selection`;
* создать событие в `palata_status_events`;
* создать action item заказчику: `experts_matched`.

Если экспертов не найдено:

* заказ оставить/перевести в `matching`;
* создать событие `no_experts_found`;
* создать action item админу: `manual_matching_required`.

# 3. Action Center для заказчика

В `/customer` добавить вкладки:

* Мои заказы
* Требуют действия
* Оценить экспертов

## Вкладка «Мои заказы»

Оставить текущий канбан.

## Вкладка «Требуют действия»

Показывать action items текущего заказчика.

### Action type: `experts_matched`

Показывать:

* заказ;
* список подобранных экспертов;
* количество экспертов;
* кнопку «Посмотреть экспертов».

При открытии списка экспертов показывать карточки:

* ФИО;
* рейтинг;
* регионы;
* направления;
* готовность к командировкам;
* регистрация в Палата судебных экспертов РФ;
* номер регистрации;
* регистрация в Центр судэксперт;
* номер регистрации;
* количество выполненных заказов;
* краткое описание опыта.

Действие:

* кнопка «Выбрать эксперта».

После выбора:

* match выбранного эксперта получает статус `selected_by_customer`;
* создаётся/обновляется запись в `palata_request_contacts`;
* заказчику открываются контакты эксперта;
* эксперту открываются контакты заказчика;
* создаётся action item эксперту: `customer_selected_you`;
* action item заказчика `experts_matched` закрывается;
* событие пишется в `palata_status_events`;
* email-event пишется в `palata_email_events` в test mode.

### Action type: `expert_declined`

Показывать:

* эксперт отказался;
* причина;
* комментарий;
* кнопку «Выбрать другого эксперта».

### Action type: `expert_can_start_from`

Показывать:

* эксперт готов начать с даты;
* дата;
* комментарий.

Действия заказчика:

* «Согласовать дату»
* «Выбрать другого эксперта»

Если «Согласовать дату»:

* заказ получает статус `in_work`;
* выбранный эксперт получает статус `accepted_work`;
* остальные эксперты получают статус `closed_by_other_expert`;
* action item закрывается;
* эксперту создаётся action item `customer_approved_start_date`;
* событие и email-event логируются.

# 4. Action Center для эксперта

В `/expert` добавить вкладки:

* Мои обращения
* Требуют действия
* Профиль эксперта
* Оценить заказчиков

## Вкладка «Мои обращения»

Оставить текущий канбан.

## Вкладка «Требуют действия»

Показывать action items текущего эксперта.

### Action type: `customer_selected_you`

Показывать:

* заказ;
* заказчика;
* регион;
* направление;
* описание;
* файлы;
* контакты заказчика, если они открыты.

Действия эксперта:

* «Взять в работу»
* «Могу начать с даты»
* «Отказаться»

Если эксперт нажал «Взять в работу»:

* заказ получает статус `in_work`;
* текущий match получает статус `accepted_work`;
* остальные matches по этому заказу получают статус `closed_by_other_expert`;
* открытые action items других экспертов по этому заказу закрываются как `cancelled`;
* action item эксперта закрывается;
* заказчику создаётся action item `expert_started_work`;
* событие и email-event логируются.

Если эксперт нажал «Могу начать с даты»:

* сохранить дату;
* match получает статус `can_start_from`;
* action item эксперта закрывается;
* заказчику создаётся action item `expert_can_start_from`;
* событие и email-event логируются.

Если эксперт нажал «Отказаться»:

* показать поле причины и комментария;
* match получает статус `declined`;
* action item эксперта закрывается;
* заказчику создаётся action item `expert_declined`;
* событие и email-event логируются.

# 5. Повторный подбор

Если все подобранные эксперты отказались:

* заказ возвращается в статус `matching`;
* `matching_round` увеличивается на 1;
* запускается новый подбор;
* эксперты, которые уже отказались, исключаются;
* создаётся новый action item заказчику `experts_matched`, если найдены новые эксперты;
* если не найдены — action item админу `manual_matching_required`.

# 6. Badges / счётчики

Вкладки должны показывать счётчики:

Для заказчика:

* `Требуют действия (N)`
* `Оценить экспертов (N)`

Для эксперта:

* `Требуют действия (N)`
* `Оценить заказчиков (N)`

N считать так:

* assigned_to_user_id = текущий пользователь;
* is_resolved = false;
* status = open;
* is_read = false или read_at is null.
