Следующий этап: реализовать выбор эксперта заказчиком из Action Center.

Сейчас уже должен быть первый кусок:

* после matching создаётся `palata_action_items`;
* заказчик видит вкладку «Требуют действия»;
* badge показывает количество открытых задач;
* action item `experts_matched` ведёт в карточку заказа.

Теперь нужно добавить выбор эксперта.

# Цель этапа

Заказчик открывает action item `experts_matched`, видит список подобранных экспертов и выбирает одного эксперта для связи.

# 1. В карточке заказа `/requests/:id`

Если текущий пользователь = customer и по заказу есть action item:

`action_type = experts_matched`
`status = open`
`is_resolved = false`

показать блок:

`Подобранные эксперты`

Для каждого эксперта из `palata_request_matches` показать карточку:

* ФИО;
* рейтинг;
* регионы работы;
* направления экспертиз;
* готовность к командировкам;
* регистрация в Палата судебных экспертов РФ;
* номер регистрации;
* регистрация в Центр судэксперт;
* номер регистрации;
* количество выполненных заказов;
* краткое описание опыта.

Кнопка:
`Выбрать эксперта`

# 2. Что происходит при выборе эксперта

При нажатии `Выбрать эксперта`:

1. В `palata_request_matches` выбранного эксперта:

   * status = `selected_by_customer`

2. В `palata_request_contacts`:

   * создать или обновить запись по request_id + expert_id
   * contact_opened_at = now()
   * expert_status = `selected_by_customer`

3. В `palata_action_items`:

   * action item заказчика `experts_matched` закрыть:

     * status = `resolved`
     * is_resolved = true
     * resolved_at = now()

4. Создать action item эксперту:

   action_type:
   `customer_selected_you`

   assigned_to_user_id:
   user_id выбранного эксперта из `palata_expert_profiles.user_id`

   title:
   `Вас выбрали по заказу`

   description:
   `Заказчик выбрал вас для связи по заказу #{номер}`

   payload:

   * request_id
   * expert_id
   * customer_id
   * contact_opened_at

5. В `palata_status_events`:
   создать событие:
   `expert_selected_by_customer`

6. В `palata_email_events`:
   создать test email event:
   `customer_selected_expert`

# 3. Контакты

После выбора:

## Заказчик видит:

* телефон эксперта;
* email эксперта;
* мессенджеры, если есть.

## Эксперт видит:

* имя заказчика;
* телефон заказчика;
* email заказчика.

Важно:
контакты показывать только если есть запись в `palata_request_contacts`.

# 4. Badge эксперта

После выбора эксперта у него во вкладке:

`Требуют действия`

должен появиться badge:

`Требуют действия (1)`

Считать по `palata_action_items`.

# 5. Важно

Пока НЕ делать:

* эксперт берёт в работу;
* эксперт отказывается;
* эксперт может с даты.

Это будет следующий этап.

Сейчас только:

* заказчик выбирает эксперта;
* контакты открываются;
* эксперту создаётся action item.
