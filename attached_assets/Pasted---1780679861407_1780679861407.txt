# Вкладка эксперта: «Метрики платформы»

Цель:
Показать эксперту:

* насколько он успешен относительно других;
* какие направления востребованы;
* какие сертификаты имеет смысл получать дополнительно.

## Блок 1. Моя позиция

### 1.1 Мой рейтинг

Формула:

Средний рейтинг эксперта =
AVG(palata_expert_ratings.score)

Источник:
palata_expert_ratings

### 1.2 Средний рейтинг по моим направлениям

Формула:

Средний рейтинг всех экспертов,
у которых есть хотя бы одно совпадающее направление с экспертом.

Источник:
palata_expert_ratings
palata_expert_certificates

### 1.3 Выполнено заказов

Формула:

COUNT(orders)

где:

status = completed

и эксперт назначен исполнителем.

Источник:
palata_requests

### 1.4 Конверсия эксперта

Формула:

Количество заказов со статусом completed
/
Количество заказов, где эксперт был выбран заказчиком
× 100%

Источник:
palata_request_matches
palata_requests

## Блок 2. ТОП-10 направлений платформы

Период:
Последние 90 дней

Формула:

COUNT(orders)

GROUP BY expertise_direction_id

ORDER BY COUNT DESC

TOP 10

Источник:
palata_requests

Показывать:

Направление
Количество заказов

---

## Блок 3. ТОП-10 направлений дистанционной работы

Период:
Последние 90 дней

Формула:

COUNT(orders)

WHERE requires_travel = false

GROUP BY expertise_direction_id

ORDER BY COUNT DESC

TOP 10

Источник:
palata_requests

---

## Блок 4. ТОП-10 направлений выездной работы в моем регионе

Регион берется из:

palata_expert_regions

Период:
Последние 90 дней

Формула:

COUNT(orders)

WHERE:
requires_travel = true
AND request.region_id IN (регионы эксперта)

GROUP BY expertise_direction_id

ORDER BY COUNT DESC

TOP 10

Источник:
palata_requests

---

## Блок 5. Упущенные возможности

Цель:
Показать эксперту направления,
по которым есть спрос,
но у него нет действующего сертификата.

Формула:

Берем ТОП направлений платформы.

Исключаем направления из:

palata_expert_certificates

где:

status = verified
AND cert_valid_to >= current_date

Оставшиеся направления показываем как рекомендации.

Источник:
palata_requests
palata_expert_certificates

---

## Блок 6. Рекомендации платформы

Формула рейтинга направления:

score =
количество заказов за 90 дней
/
количество активных экспертов по направлению

где активный эксперт:

есть verified сертификат
cert_valid_to >= current_date

Сортировка:

score DESC

TOP 5

Показывать:

Направление
Количество заказов
Количество активных экспертов
Индекс дефицита

Пример:

Товароведческая
38 заказов
5 экспертов
Индекс дефицита = 7.6

Строительная
22 заказа
2 эксперта
Индекс дефицита = 11

Чем выше индекс дефицита,
тем выгоднее получать данный сертификат.
