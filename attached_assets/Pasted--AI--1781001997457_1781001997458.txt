КРИТИЧЕСКАЯ ЗАДАЧА. Нужно встроить AI-определение направления экспертизы при создании заказа.

Кредиты и сроки ограничены. Нужна точечная реализация без архитектурных экспериментов.

Исходная логика:
Сейчас при создании заказа поле "Направление экспертизы" выбирает заказчик вручную.
Нужно изменить логику.

Новая логика:

1. На форме создания заказа заказчик сначала НЕ видит поле "Направление экспертизы".

2. Заказчик заполняет описание ситуации в свободной форме.

3. После заполнения описания система вызывает AI-помощника.

4. AI-помощник должен:

   * проанализировать описание ситуации;
   * пройтись по базе знаний;
   * определить направление экспертизы;
   * выбрать направление ТОЛЬКО из справочника public.palata_expertise_directions.

5. Если AI уверенно определил направление:

   * направление автоматически проставляется в заказ;
   * заказ создаётся;
   * в palata_requests.expertise_direction_id записывается найденное направление.

6. Если AI НЕ смог определить направление:

   * заказ НЕ создаётся;
   * пользователю показать сообщение:

"По вашему заказу мы не смогли определить направление экспертизы. Вы можете самостоятельно выбрать направление из предложенного списка."

* после этого показать поле "Направление экспертизы";
* заказчик выбирает направление вручную;
* только после выбора направления заказ можно создать.

7. Заказ НЕ может быть создан без expertise_direction_id.

---

ВАЖНО ПО БЕЗОПАСНОСТИ

OpenAI API key нельзя хранить на фронте.

Нельзя использовать VITE_OPENAI_API_KEY.

API key должен быть только на backend/serverless/backend route.

Если текущий frontend на Vercel не имеет backend-роута, предложить минимальный безопасный вариант:

* либо serverless endpoint /api/ai-detect-direction;
* либо существующий api-server;
* либо Supabase Edge Function.

Но ключ OpenAI не должен попасть в браузер.

---

ENV

Нужно добавить переменную окружения:

OPENAI_API_KEY

Только для backend/serverless окружения.

---

База знаний

В проект нужно добавить/использовать файл базы знаний:
"База знаний для ИИ-подбора экспертов"

Смысл базы:

* AI не ведёт диалог;
* AI не задаёт вопросы;
* AI не пишет развернутый ответ пользователю;
* AI только классифицирует описание ситуации;
* результат должен быть машинно-читаемым.

---

Формат ответа AI строго JSON:

{
"detected": true,
"direction_name": "Судебно-строительная экспертиза",
"confidence": 0.87,
"reason": "Краткое техническое основание",
"matched_markers": ["залив", "повреждение ремонта", "ущерб"]
}

Если направление не определено:

{
"detected": false,
"direction_name": null,
"confidence": 0,
"reason": "Недостаточно признаков для выбора направления",
"matched_markers": []
}

---

Критично:

AI не имеет права возвращать направление, которого нет в public.palata_expertise_directions.

Алгоритм должен быть такой:

1. Получить список направлений из public.palata_expertise_directions.
2. Передать AI только допустимый список direction_name.
3. AI выбирает только из этого списка.
4. После ответа AI найти direction_id по direction_name.
5. Если direction_id не найден — считать, что направление не определено.

---

Минимальный порог уверенности:

Если confidence < 0.7:

* считать, что направление не определено;
* показать ручной выбор направления.

---

UI-логика в NewRequest.tsx

1. Поле "Описание ситуации" остаётся обязательным.

2. Поле "Направление экспертизы":

   * по умолчанию скрыто;
   * показывается только если AI не смог определить направление;
   * если показано, становится обязательным.

3. Кнопка создания заказа:

При первом нажатии:

* если направление ещё не определено;
* вызвать AI;
* если AI определил направление — создать заказ;
* если AI не определил — показать сообщение и поле ручного выбора.

При повторном нажатии после ручного выбора:

* создать заказ с выбранным direction_id.

---

Пользовательские сообщения

Пока AI анализирует:

"Определяем направление экспертизы по описанию ситуации..."

Если AI определил:

"Направление экспертизы определено автоматически: <direction_name>"

Если AI не определил:

"По вашему заказу мы не смогли определить направление экспертизы. Вы можете самостоятельно выбрать направление из предложенного списка."

Если ошибка AI/API:

"Не удалось автоматически определить направление экспертизы. Выберите направление вручную."

При ошибке AI заказ НЕ создавать без направления.

---

Промт для AI

Системная роль:
Ты классификатор направления экспертизы. Не веди диалог. Не задавай вопросов. Не объясняй пользователю. Верни только JSON.

Задача:
По описанию ситуации определить направление экспертизы из списка допустимых направлений.
Используй базу знаний как источник маркеров и сценариев.
Если уверенности недостаточно — верни detected=false.
Нельзя придумывать направления.
Нельзя выбирать направление вне справочника.

---

Проверки перед созданием заказа

Перед insert в palata_requests:

if (!expertise_direction_id) {
throw / показать ошибку;
}

Заказ без направления создавать запрещено.

---

Acceptance criteria

1. Пользователь описал:
   "Нас залили соседи, испорчен ремонт, нужно посчитать ущерб"

AI должен определить:
Судебно-строительная экспертиза

Заказ создаётся с expertise_direction_id.

2. Пользователь описал:
   "Нужно проверить качество ремонта, плитка отваливается"

AI должен определить:
Судебно-строительная экспертиза

3. Пользователь описал:
   "Нужно понять что делать по спору"

AI не должен угадывать.
Показать ручной выбор направления.

4. Если AI вернул направление, которого нет в справочнике:
   заказ не создавать;
   показать ручной выбор.

5. Если OpenAI API недоступен:
   заказ не создавать;
   показать ручной выбор.

6. Если заказчик вручную выбрал направление:
   заказ создаётся корректно.

7. В OpenAI API key нет утечки во frontend bundle.
