Нужно добавить безопасное восстановление пароля через Supabase Auth.

SMTP через Resend уже настроен.

1. На странице входа

Если пользователь ввёл неверный пароль, показывать:

"Неверный email или пароль"

Под сообщением добавить ссылку/кнопку:

"Восстановить пароль"

2. Восстановление пароля

При нажатии открыть форму:

- email
- кнопка "Отправить письмо для восстановления"

При отправке вызвать:

supabase.auth.resetPasswordForEmail(email, {
  redirectTo: window.location.origin + "/reset-password"
})

После успешного вызова показать сообщение:

"Если пользователь с такой почтой зарегистрирован, мы отправили письмо для восстановления пароля."

Важно:
Не раскрывать, существует ли email в системе.

3. Страница /reset-password

Создать страницу /reset-password.

Логика:

- обработать сессию восстановления пароля от Supabase;
- показать форму:
  - новый пароль
  - повторите новый пароль
- проверить:
  - оба пароля совпадают
  - пароль не короче минимального требования проекта

При сохранении вызвать:

supabase.auth.updateUser({
  password: newPassword
})

После успешной смены пароля показать:

"Пароль успешно изменён. Теперь вы можете войти в систему."

И перенаправить на /login.

4. Нельзя

- Нельзя отправлять пароль в письме.
- Нельзя хранить пароль в public.palata_users или других таблицах.
- Нельзя показывать старый пароль.
- Нельзя создавать свой механизм хранения паролей.
- Нельзя логировать пароль в console.

5. Шаблон письма Supabase

Использовать стандартный шаблон Recovery / Reset password.

Тема письма:

"Восстановление пароля на платформе судебных экспертов"

Текст письма:

"Здравствуйте!

Мы получили запрос на восстановление пароля для входа на платформу судебных экспертов.

Чтобы задать новый пароль, нажмите кнопку ниже.

[Сменить пароль]

Если вы не запрашивали восстановление пароля, просто проигнорируйте это письмо."

Кнопка должна вести на {{ .ConfirmationURL }}.
