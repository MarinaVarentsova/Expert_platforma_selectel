Нужно исправить две проблемы в основном приложении Expert_platforma_selectel.

ЗАДАЧА 1. UX страницы reset-password

Страница уже существует в основном приложении и успешно вызывает auth-service.

Сейчас при ошибке пользователь видит:

HTTP 400

Нужно:

1. Найти страницу /reset-password.
2. При ответе API читать JSON.
3. Показывать поле error из ответа.
4. Не показывать HTTP 400, HTTP 500, Bad Request.
5. Для недействительного токена показывать:

«Ссылка восстановления недействительна или уже была использована. Откройте самое последнее письмо или запросите восстановление пароля повторно.»

6. Для неизвестной ошибки:

«Произошла ошибка. Попробуйте ещё раз.»

7. Во время запроса блокировать кнопку и показывать загрузку.
8. После успеха показать «Пароль успешно изменён» и через 2–3 секунды перенаправить на вход.

ЗАДАЧА 2. При регистрации эксперта не сохраняются связанные данные

Эксперт при регистрации заполняет:

- регион;
- номер сертификата;
- направления экспертизы;
- другие поля сертификата.

Но после регистрации отсутствуют записи в:

- palata_expert_regions;
- palata_expert_certificates;
- palata_expert_directions.

Из-за этого в личном кабинете нет региона, сертификата и направлений.

Нужно провести аудит полного потока:

форма регистрации
→ payload
→ API регистрации
→ создание palata_expert_profiles
→ запись региона
→ запись сертификата
→ запись направлений
→ загрузка данных в ЛК.

Сначала ничего не менять.

Сначала показать:

1. Где находится форма регистрации эксперта.
2. Какой endpoint она вызывает.
3. Какой payload отправляет.
4. Где создаётся palata_expert_profiles.
5. Почему не создаются записи в palata_expert_regions, palata_expert_certificates и palata_expert_directions.
6. Какие файлы будут изменены.

После анализа исправить:

- сохранение региона;
- сохранение сертификата;
- сохранение направлений;
- правильный expert_id;
- защиту от дублей;
- загрузку и отображение данных в ЛК;
- обновление через «Редактировать профиль».

В конце вернуть:

- точную причину;
- список изменённых файлов;
- SQL для проверки;
- сценарий ручного тестирования;
- действия для redeploy в Coolify.
