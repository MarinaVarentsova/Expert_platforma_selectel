# DEF-01 Critical

## Кнопка «Выбрать эксперта» не должна зависеть от open action item

Сейчас кнопка «Выбрать эксперта» появляется только если есть открытый `experts_matched` action item.

Проблема:
если action item уже закрыт или не был создан, но заказ всё ещё находится в статусе `expert_selection`, заказчик не может выбрать эксперта.

Нужно изменить логику показа кнопки.

Кнопка «Выбрать эксперта» должна отображаться, если:

* текущий пользователь = заказчик этого заказа;
* request.status = `expert_selection`;
* есть доступные `palata_request_matches`;
* match эксперта ещё не закрыт:

  * не `declined`;
  * не `closed_by_other_expert`;
  * не `withdrawn`;
  * не `accepted_work`;
  * не `completed`.

Не привязывать кнопку только к существованию open action item `experts_matched`.

Action item может быть дополнительным входом в сценарий, но не единственным условием для выбора эксперта.

# DEF-02 High

## Админ должен иметь возможность запустить подбор для заказа в статусе matching

Сейчас кнопка «Запустить подбор» не показывается админу для заказа в статусе `matching`.

Нужно добавить кнопку для admin:

`Запустить подбор экспертов`

Показывать, если:

* role = `admin`;
* request.status IN (`new`, `pending`, `matching`);
* заказ ещё не completed;
* заказ не not_actual / cancelled.

При нажатии:

* вызвать существующий `runMatching()`;
* не создавать дубли matches, если они уже есть;
* учитывать уже отказавшихся экспертов;
* создать событие в `palata_status_events`;
* если эксперты найдены:

  * request.status = `expert_selection`;
  * создать/обновить action item заказчику `experts_matched`;
* если эксперты не найдены:

  * оставить/вернуть request.status = `matching`;
  * создать action item админу `manual_matching_required`.
