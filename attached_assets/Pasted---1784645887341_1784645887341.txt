ЗАДАЧА

1. Проверить текущую реализацию новых endpoint'ов.

Не переписывать её без найденной ошибки.

Проверить:

- корректно ли определяется user_id созданного пользователя;
- совпадает ли user_id с expert_id, который ожидают таблицы;
- используются ли реальные названия колонок PostgreSQL;
- выполняется ли транзакция;
- не создаются ли дубли;
- возвращают ли endpoint'ы понятный JSON при успехе и ошибке.

2. Проверить последовательность в Register.tsx.

После успешного создания пользователя должны последовательно выполниться:

- сохранение профиля эксперта;
- сохранение регионов;
- сохранение направлений;
- сохранение сертификата.

Не считать регистрацию данных эксперта успешной, пока обязательные записи не сохранены.

При ошибке показать понятный текст и залогировать, на каком шаге произошла ошибка.

Не логировать пароль, токены и персональные данные.

3. Проверить загрузку данных в личном кабинете.

Найти API и код страницы профиля эксперта, которые загружают:

- palata_expert_profiles;
- palata_expert_regions;
- palata_expert_certificates;
- palata_expert_directions.

Проверить, что после первого входа API возвращает:

- регион;
- номер сертификата;
- срок действия сертификата;
- направления экспертизы.

4. Проверить соответствие структуры ответа API структуре, которую ожидает frontend.

Частые варианты ошибки:

- API возвращает region_id, а frontend ждёт region;
- API возвращает certificates массивом, а frontend ждёт certificate;
- API возвращает direction_id, а frontend ждёт direction_name;
- связи созданы по user_id, а чтение выполняется по profile_id;
- expert_id в таблицах не совпадает с идентификатором текущего пользователя.

Исправить только найденное несоответствие.

5. После успешной регистрации и входа данные должны отображаться сразу, без нажатия «Редактировать профиль».

6. Редактирование профиля должно продолжить работать как раньше.

Не ломать существующие endpoint'ы редактирования.

- одним регионом;
- одним сертификатом;
- минимум одним направлением.

После регистрации проверить SQL:

WITH u AS (
  SELECT id
  FROM palata_users
  WHERE email = 'EMAIL_НОВОГО_ЭКСПЕРТА'
  LIMIT 1
)
SELECT
  (SELECT COUNT(*)
   FROM palata_expert_profiles
   WHERE user_id = (SELECT id FROM u)) AS profiles,

  (SELECT COUNT(*)
   FROM palata_expert_regions
   WHERE expert_id = (SELECT id FROM u)) AS regions,

  (SELECT COUNT(*)
   FROM palata_expert_directions
   WHERE expert_id = (SELECT id FROM u)) AS directions,

  (SELECT COUNT(*)
   FROM palata_expert_certificates
   WHERE expert_id = (SELECT id FROM u)) AS certificates;

Ожидается:

profiles = 1
regions >= 1
directions >= 1
certificates >= 1

Затем войти этим пользователем и проверить, что в ЛК сразу отображаются:

- регион;
- сертификат;
- направления.
