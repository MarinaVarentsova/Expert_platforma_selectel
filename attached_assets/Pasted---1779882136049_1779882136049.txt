Нужно привести в порядок адаптивную верстку проекта «Палата судебных экспертов».

Сейчас на широком экране канбан и страницы отображаются некорректно:

* контент прижат к левому краю;
* справа остаётся огромное пустое поле;
* колонки канбана не используют ширину экрана;
* часть карточек уходит вниз/обрезается;
* нет нормальной адаптации под разные размеры экрана.

Нужно сделать полноценный responsive layout для всех основных страниц:

* `/customer`
* `/expert`
* `/admin`
* `/admin/metrics`
* `/requests/:id`
* `/login`
* главная страница

## 1. Общий layout

Сделать единый `AppShell` / layout:

* `max-width` для контента на больших экранах;
* центрирование контента;
* нормальные внутренние отступы;
* адаптивные padding:

  * desktop: 32–48px;
  * tablet: 24px;
  * mobile: 16px.

Контент не должен висеть узкой колонкой слева.

## 2. Канбан-доски

Переделать kanban layout:

### Desktop

* колонки должны равномерно занимать ширину контейнера;
* если колонок много — использовать горизонтальный scroll только внутри блока канбана;
* высота колонок должна быть аккуратной;
* карточки не должны обрезаться.

### Tablet

* 2–3 колонки в ряд или горизонтальный scroll;
* карточки остаются читаемыми.

### Mobile

* колонки идут вертикально;
* каждая колонка занимает 100% ширины;
* без горизонтального развала страницы.

## 3. Header / navigation

Исправить верхнюю навигацию:

* логотип слева;
* ролевые вкладки не должны ломать ширину;
* профиль пользователя справа;
* на маленьких экранах меню должно схлопываться или переноситься аккуратно;
* не должно быть горизонтального скролла всей страницы.

## 4. Карточки

Привести карточки к единому виду:

* одинаковая ширина внутри колонок;
* нормальные отступы;
* аккуратная высота;
* текст не должен вылезать;
* длинные названия обрезать через line-clamp;
* добавить hover-state.

## 5. Метрики

На `/admin/metrics`:

* KPI-карточки должны адаптироваться сеткой:

  * desktop: 4 карточки в ряд;
  * tablet: 2 карточки;
  * mobile: 1 карточка;
* правая панель на маленьких экранах должна уходить ниже основного контента;
* не должно быть обрезания и пустых огромных зон.

## 6. Детальная карточка заказа

На `/requests/:id`:

* сделать аккуратную двухколоночную структуру на desktop;
* на tablet/mobile переводить в одну колонку;
* блоки: данные заказа, эксперты, события, действия — должны быть читаемыми.

## 7. Mobile-first проверка

Проверить размеры:

* 1440px
* 1280px
* 1024px
* 768px
* 390px

## 8. Важно

* не менять бизнес-логику;
* не ломать Supabase-запросы;
* не менять статусы;
* править только layout, CSS, компоненты обёртки и responsive behavior;
* сохранить текущий тёплый enterprise-style;
* после работы показать, какие компоненты layout были изменены.
