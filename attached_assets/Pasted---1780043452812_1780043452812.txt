Следующий этап: реализовать завершение заказа экспертом и оценку заказчика.

Сейчас уже должно работать:

* заказчик выбирает эксперта;
* эксперт может взять заказ в работу;
* эксперт может предложить дату;
* заказчик может согласовать дату;
* заказ переходит в `in_work`.

Теперь нужно реализовать завершение заказа.

# Цель этапа

Эксперт по заказу в статусе `in_work` должен иметь возможность завершить заказ и сразу оценить заказчика.

Заказчик НЕ должен переводить заказ в выполненные самостоятельно.

# 1. В ЛК эксперта

В карточке заказа, если:

* текущий пользователь = expert;
* заказ status = `in_work`;
* текущий эксперт по заказу имеет match status = `accepted_work`;

показать кнопку:

`Завершить заказ`

# 2. Форма завершения

При нажатии показать форму:

Поля:

* оценка заказчика от 1 до 5;
* комментарий эксперта;
* итоговый комментарий по заказу.

Кнопка:
`Завершить и оценить заказчика`

# 3. Что происходит после завершения

При сохранении:

## palata_requests

* status = `completed`
* updated_at = now()

## palata_request_matches

Для текущего эксперта:

* status = `completed`

## palata_request_contacts

Для текущего эксперта:

* expert_status = `completed`
* expert_status_updated_at = now()
* final_result = `completed`
* expert_comment = итоговый комментарий

## palata_customer_ratings

Создать запись:

* request_id
* expert_id
* customer_id
* score
* comment
* created_at

## palata_status_events

Создать событие:
`expert_completed_order`

event_payload:

* request_id
* expert_id
* customer_id
* score
* comment

## palata_email_events

Создать test email event:
`expert_completed_order`

# 4. Создать action item заказчику

После завершения заказа экспертом создать action item заказчику:

action_type:
`expert_completed_order`

title:
`Эксперт завершил заказ`

description:
`Эксперт завершил работу по заказу #{номер}. Пожалуйста, оцените эксперта.`

assigned_to_user_id:
customer user id

payload:

* request_id
* expert_id
* completed_at

# 5. В ЛК заказчика

Во вкладке:

`Оценить экспертов`

показывать заказы, где:

* заказ status = `completed`;
* есть action item `expert_completed_order`;
* is_resolved = false;
* оценка эксперта заказчиком ещё не создана в `palata_expert_ratings`.

Карточка должна показывать:

* номер заказа;
* название;
* эксперт;
* дата завершения;
* кнопку `Оценить эксперта`.

# 6. Важно

* заказчик не может переводить заказ в `completed`;
* заказчик только оценивает эксперта после завершения экспертом;
* все действия писать в Supabase;
* всё логировать в `palata_status_events`;
* email пока только test mode;
* не использовать моковые данные.

4. почему заказчик больше не может сам завершить заказ;
5. как заказ появляется во вкладке «Оценить экспертов».
