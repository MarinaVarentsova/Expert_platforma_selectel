Отчёт аудита matching-логики
1. Где находится основной matching-код
artifacts/palata/src/lib/matching.ts

2. Поле формата работы
palata_requests.requires_travel: boolean — true = выезд, false = дистанционно

3–6. Логика по сценариям (после исправления)
Сценарий	Условие	Действие
1	direction + region + выезд	Фильтр: регион совпадает + business_trip_ready = true; сортировка по рейтингу
2	direction + region + дистанционно	Фильтр только по направлению; регион игнорируется полностью
3	нет direction	Ранний выход → manual_matching_required для администратора
4	нет region + direction + дистанционно	Фильтр только по направлению (как сценарий 2)
5	нет region + выезд	Ранний выход → manual_matching_required для администратора
7. Рейтинг
Первичная сортировка: avg_customer_rating (null → 0), 10 очков за каждую единицу рейтинга. Тайбрейкеры — верификация и выполненные заказы. Чистый формульный score возвращается в MatchingResult.experts.
