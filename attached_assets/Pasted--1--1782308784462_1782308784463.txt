Уточняю баг. Нужно исправить именно сценарий старой страницы у заказчика.

Сценарий:

1. Заказчик открыл страницу заказа.
2. На странице видит 2 предложенных экспертов.
3. По эксперту 1 заказчик нажал "Выбрать эксперта".
4. Эксперт 1 взял заказ в работу.
5. Заказчик НЕ обновляет страницу.
6. На старой странице у заказчика всё ещё видна кнопка "Выбрать эксперта" по эксперту 2.
7. Заказчик нажимает "Выбрать эксперта" по эксперту 2.
8. Сейчас эксперту 2 уходит уведомление, хотя заказ уже в работе у эксперта 1.

Требуемое поведение:

При нажатии "Выбрать эксперта" по эксперту 2 нужно прямо перед отправкой уведомления и любыми update сделать свежую проверку в БД:

* есть ли по этому request_id уже эксперт в работе.

Если есть:

* не отправлять уведомление эксперту 2;
* не менять статус эксперта 2;
* не создавать/не обновлять contacts для эксперта 2;
* показать заказчику сообщение:
  "На заказ уже назначен эксперт";
* обновить данные заказа на странице.

Проверка должна стоять в обработчике кнопки "Выбрать эксперта" у заказчика, а не только при загрузке страницы.

Важно:
Проверка должна идти ДО:

* отправки email/notification;
* изменения palata_request_matches;
* изменения palata_request_contacts;
* открытия следующего шага.

Пример логики:

handleChooseExpert(expert2):

1. fresh select из palata_request_matches:
   where request_id = current request_id
   and status = 'in_work' / фактический статус "в работе"

2. Если найден хотя бы один match:
   showMessage("На заказ уже назначен эксперт")
   refreshRequest()
   return

3. Если не найден:
   выполнять текущую логику выбора эксперта 2.

Нужно найти все обработчики кнопок заказчика:

* "Выбрать эксперта"
* "Предложить эксперту"
* любые похожие действия, которые отправляют уведомление эксперту

и поставить туда одну общую проверку.

Acceptance test:

1. Заказчик открывает заказ и видит 2 экспертов.
2. Выбирает эксперта 1.
3. Эксперт 1 берет заказ в работу.
4. Заказчик НЕ обновляет страницу.
5. Заказчик нажимает "Выбрать эксперта" по эксперту 2.
6. Ожидаем:

   * появляется сообщение "На заказ уже назначен эксперт";
   * уведомление эксперту 2 НЕ уходит;
   * статус эксперта 2 НЕ меняется;
   * данные страницы обновляются.

Вернуть:

1. Где находится обработчик кнопки "Выбрать эксперта" у заказчика.
2. Где добавлена fresh DB check.
3. Какой статус считается "эксперт уже в работе".
4. Подтверждение, что уведомление эксперту 2 блокируется до отправки.
5. Логи/SQL проверки до и после.
