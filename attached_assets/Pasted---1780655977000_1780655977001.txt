Нужно добавить новую вкладку для эксперта: "Рынок".

Логика:

Если по заказу автоподбор не нашёл экспертов, заказ НЕ пропадает.
Он должен попадать на общий рынок заказов для экспертов.

У заказчика ничего не меняется:
- заказ остаётся в статусе "идёт подбор экспертов";
- заказчик не видит отдельной логики рынка.

# 1. Новая вкладка эксперта

В ЛК эксперта добавить вкладку:

"Рынок"

В ней показывать заказы, по которым:
- статус = идёт подбор экспертов / matching / matching_started / expert_selection — использовать текущий фактический статус системы;
- нет активных подобранных экспертов в palata_request_matches;
- заказ не завершён;
- заказ не отменён;
- заказ не not_actual.

# 2. Что показывать в карточке заказа

Для каждого заказа:

- название заказа;
- заказчик;
- регион заказа;
- направление экспертизы;
- формат работы: выезд / дистанционно;
- описание;
- дата создания;
- рейтинг заказчика;
- статус заказа.

# 3. Фильтры рынка

Добавить фильтры:

- направление;
- регион;
- дистанционно;
- выезд.

Фильтры работают по:

- palata_requests.expertise_direction_id;
- palata_requests.region_id;
- palata_requests.requires_travel.

# 4. Сортировка

Сортировка по рейтингу заказчика:

- сначала заказчики с более высоким рейтингом;
- если рейтинга нет — считать рейтинг 0.

Источник рейтинга:
palata_customer_ratings.score.

# 5. Действие эксперта

Эксперт может открыть карточку заказа и нажать:

"Могу взять с ..."

где выбирает дату, с которой готов приступить.

После нажатия использовать уже реализованную схему обработки назначения эксперта:

- создать/обновить запись в palata_request_matches;
- эксперт становится кандидатом на заказ;
- заказчик должен увидеть эксперта в списке подобранных / откликнувшихся экспертов;
- дальше работает уже существующий сценарий выбора эксперта заказчиком;
- не ломать текущие статусы и action items.

# 6. Важно

Эксперт видит на рынке только те заказы, по которым он потенциально может работать:

Для дистанционного заказа:
- совпадает направление с действующим verified сертификатом эксперта;
- сертификат не истёк;
- accepts_requests = true.

Для выездного заказа:
- совпадает направление с действующим verified сертификатом;
- сертификат не истёк;
- accepts_requests = true;
- business_trip_ready = true;
- регион заказа совпадает с одним из регионов эксперта в palata_expert_regions.

# 7. Не использовать legacy

Не использовать:

- palata_request_regions;
- palata_requests.region;
- palata_expert_profiles.regions;
- palata_expert_directions как основной источник допуска;
- palata_expert_profiles.specializations.

Источник допуска эксперта:
palata_expert_certificates:
- status = verified;
- cert_valid_to >= current_date;
- cert_direction_ids содержит expertise_direction_id заказа.

# 8. Если эксперт уже откликался

Если эксперт уже есть в palata_request_matches по этому заказу:
- не показывать кнопку "Могу взять";
- показать статус отклика.

# 9. Отчёт после реализации

6. как заказчик увидит эксперта после отклика;
