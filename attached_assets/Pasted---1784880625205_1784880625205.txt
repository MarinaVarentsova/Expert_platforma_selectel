ТЕКУЩАЯ СИТУАЦИЯ

Но production в Coolify запускает:

artifacts/palata/server.js

Поэтому этот dev-маршрут в production не используется, а POST /api/ai-detect-direction возвращает 404.

1. Найти последний commit, в котором были добавлены только диагностические логи в:
   artifacts/api-server/src/routes/ai-detect-direction.ts

2. Удалить только эти добавленные логи и вернуть файл к состоянию до этого commit.

3. Не удалять:
   - сам маршрут;
   - KNOWLEDGE_BASE;
   - бизнес-логику AI;
   - типы;
   - обработку ошибок.

4. Показать diff отката этого файла.

ЗАДАЧА 2 — добавить production-маршрут правильно

1. Найти в artifacts/palata/server.js место, где разбираются входящие API-маршруты.

2. Добавить обработку:

POST /api/ai-detect-direction

именно в production-server artifacts/palata/server.js.

3. Не копировать KNOWLEDGE_BASE вручную в server.js.

4. Вынести общую AI-логику и KNOWLEDGE_BASE в один общий модуль, который смогут использовать:
   - artifacts/api-server/src/routes/ai-detect-direction.ts
   - artifacts/palata/server.js

Либо использовать уже существующий общий модуль, если он есть.

5. Сохранить текущий JSON-контракт frontend:
   - те же входные поля;
   - те же коды ответа;
   - тот же формат success/error;
   - те же поля detected, direction_name, confidence, reason и matched direction.

6. Production endpoint должен:
   - принимать description;
   - принимать availableDirections;
   - проверять OPENAI_API_KEY;
   - использовать существующую KNOWLEDGE_BASE;
   - обращаться к OpenAI;
   - сопоставлять ответ только с approved availableDirections;
   - не создавать заказ и не менять БД.

ЗАДАЧА 3 — production-логи

Добавить логи только в реально исполняемый production-путь:

Не логировать:
- OPENAI_API_KEY;
- полный текст заказа;
- полный ответ с персональными данными;
- токены;
- пароли.

Для description логировать только длину, без preview.
