Нужно встроить сценарий подтверждения email при регистрации через Supabase Auth.

Инфраструктура уже настроена:
- Supabase SMTP подключен через Resend
- Confirm email включен
- Site URL: https://expertplatforma.vercel.app
- Redirect URL: https://expertplatforma.vercel.app/**

Задача по коду:

1. В форме регистрации использовать только supabase.auth.signUp().

2. При signUp передавать:
   email
   password
   options.data = metadata пользователя
   options.emailRedirectTo = window.location.origin + "/auth/callback"

3. После успешного signUp:

   Если data.session === null:
   НЕ отправлять пользователя сразу в ЛК.

   Показать экран/сообщение:

   "Спасибо за регистрацию.
   На почту <email> отправлено письмо для подтверждения регистрации.
   Перейдите по ссылке из письма, чтобы завершить регистрацию."

4. Если data.session !== null:
   использовать существующую логику входа и перехода в ЛК.

5. Создать или проверить маршрут /auth/callback.

   На /auth/callback:
   - дождаться получения session от Supabase;
   - получить текущего пользователя;
   - найти роль пользователя в public.palata_users;
   - перенаправить:
     customer → личный кабинет заказчика
     expert → личный кабинет эксперта
     admin → личный кабинет администратора.

6. Если пользователь с такой почтой уже существует:
   показать понятное сообщение:

   "Пользователь с данной почтой уже зарегистрирован. Войдите в систему или восстановите пароль."

7. Не показывать пользователю технические ошибки Supabase.
   Технические ошибки можно логировать в console.

8. Проверить, что metadata при регистрации сохраняется:
   - role
   - full_name
   - phone
   - для заказчика: company_name, inn, contact_name, notes, region_id
   - для эксперта: certificate data, region_ids, business_trip_ready и остальные текущие поля.

Вернуть:
- какие файлы изменены;
- где находится /auth/callback;
- какой текст показывается пользователю после регистрации;
- как определяется роль после подтверждения email.
