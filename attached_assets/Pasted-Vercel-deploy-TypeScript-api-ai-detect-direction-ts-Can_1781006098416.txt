Vercel deploy падает на TypeScript в api/ai-detect-direction.ts.

Ошибки:

* Cannot find name 'Request'
* Cannot find name 'Response'
* Cannot find name 'process'
* Cannot find name 'fetch'

OPENAI_API_KEY в Vercel уже создана. Проблема НЕ в переменной, а в типах и формате serverless-функции.

Нужно срочно исправить ai-detect-direction и довести до рабочего состояния.

ТРЕБОВАНИЯ:

1. Исправить api/ai-detect-direction.ts под Vercel Serverless Function.

Использовать:

import type { VercelRequest, VercelResponse } from '@vercel/node';

export default async function handler(
req: VercelRequest,
res: VercelResponse
) {
...
}

2. Не использовать глобальные Request/Response.

3. Не использовать new Response().

Использовать:

return res.status(200).json(...)
return res.status(400).json(...)
return res.status(405).json(...)
return res.status(500).json(...)

4. Проверить наличие:

const apiKey = process.env.OPENAI_API_KEY

Ключ должен использоваться только на сервере.

Никаких VITE_OPENAI_* переменных.

5. Исправить tsconfig так, чтобы сборка проходила на Vercel.

6. После исправления выполнить локальную сборку и typecheck.

ОТДЕЛЬНО ПО ИИ-ПОМОЩНИКУ

Нужно не просто подключить OpenAI, а полностью реализовать подбор направления экспертизы.

Логика:

1. Пользователь сначала описывает ситуацию.

2. Поле "Направление экспертизы" изначально скрыто.

3. После ввода описания вызывается AI endpoint.

4. ИИ анализирует описание и определяет направление ТОЛЬКО из справочника направлений.

5. ИИ НЕ ИМЕЕТ ПРАВА придумывать свои направления.

6. Если найдено направление:

* автоматически подставить его в заказ;
* показать пользователю:
  "Определено направление: <название>";
* заказ может быть создан.

7. Если направление не определено:

показать пользователю:

"По вашему описанию мы не смогли определить направление экспертизы. Пожалуйста, выберите направление вручную."

После этого раскрыть обязательный список направлений.

8. Заказ НЕ МОЖЕТ быть создан без направления.
