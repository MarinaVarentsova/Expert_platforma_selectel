# Правильная бизнес-логика подбора

## Сценарий 1

Если в заказе указано:

* region_id
* expertise_direction_id
* формат работы = выезд

то подбираем экспертов:

1. по совпадению региона:
   palata_requests.region_id = palata_expert_regions.region_id

2. только тех, кто готов к выезду:
   palata_expert_profiles.business_trip_ready = true

3. по совпадению направления:
   palata_requests.expertise_direction_id = palata_expert_directions.expertise_direction_id

4. сортировка:
   сначала эксперты с более высоким рейтингом,
   от высокого к низкому.

---

## Сценарий 2

Если в заказе указано:

* region_id
* expertise_direction_id
* формат работы = дистанционная работа

то регион НЕ используется как фильтр.

Подбираем экспертов:

1. только по направлению:
   palata_requests.expertise_direction_id = palata_expert_directions.expertise_direction_id

2. сортировка:
   рейтинг от высокого к низкому.

---

## Сценарий 3

Если в заказе НЕ указано направление:

* expertise_direction_id is null

то экспертов НЕ подбирать.

Нужно вернуть пустой список matches и/или action item для администратора:
manual_matching_required.

---

## Сценарий 4

Если в заказе НЕ указан регион, но указано направление и формат работы = дистанционная работа:

то подбираем экспертов:

1. по направлению:
   palata_requests.expertise_direction_id = palata_expert_directions.expertise_direction_id

2. сортировка:
   рейтинг от высокого к низкому.

Регион в этом сценарии НЕ нужен.

---

## Сценарий 5

Если в заказе НЕ указан регион, но формат работы = выезд:

то экспертов НЕ подбирать.

Причина:
для выезда нужен регион заказа.

Нужно вернуть пустой список matches и/или action item для администратора:
manual_matching_required.

# Дополнительные обязательные условия

Эксперт может участвовать в подборе только если:

1. palata_expert_profiles.accepts_requests = true
2. есть хотя бы один действующий/подтверждённый сертификат:
   palata_expert_certificates.status = 'verified'
3. есть хотя бы одно направление в:
   palata_expert_directions
4. если заказ с выездом:
   business_trip_ready = true
5. если заказ с выездом:
   есть совпадение региона через palata_expert_regions
